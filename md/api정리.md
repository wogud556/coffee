## 구현이 필요한 api정리
- 유저
  - 회원가입
  - 
- 주문
  - 메뉴 전체 호출(구현)
    - /API/order
  - 메뉴 단건 호출
    - /API/order
  - 분류에 따른 메뉴 호출
  - 주문 기본
- 이벤트

### 고민이 필요한 사항
- 하나의 컨트롤러내 한개 메소드에서만 처리를 해야하는가
- 한개의 컨트롤러에서 다른 메소드에서 처리를 해야하는가

### 구분 값에 대한 설명
- 메뉴id(커피브랜드 2 + 메뉴뷴류3 + 숫자 3자리)
- 커피브랜드코드(커피 브랜드의 고유값 4자리 ex) STBS = 스타벅스, MGCF = 메가커피)
- 핫 아이스 구분 (H : only hot, I : only Ice, H/I 둘다)
- 가능 사이즈
  - S : Short
  - T : Tall
  - G : Grande
  - V : Venti
  - R : Trenta
  - 7 : 7oz
  - O : Solo(원샷)
  - D : Doppio(더블샷)
